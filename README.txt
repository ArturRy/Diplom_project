Дипломный проект профессии «Python-разработчик: расширенный курс»
Backend-приложение для автоматизации закупок

Инструкция

USER
1. Необходимо создать пользователя(user/register). При положительном запросе придет токен
2. Затем подтвердить статус пользователя (user/register/confirm) статус измениться на is_active=True
3. Затем необходимо выполнить авторизацию при помощи email и password (user/login)для получения токена аутентификации
его мы и будем использовать при  дальнейшей работе с этим пользователем
4. Пользователь может изменить пароль от аккаунта(user/password_reset , user/password_reset/confirm)
5. Можно добавить и редактировать информацию о пользователе (user/info)

PARTNER
1. Если type пользователя shop: можно обновить или создать прайс (shop/update)
2. Также можно изменить статус заказа пользователя (order), пользователю будет приходить уведомление на email при изменении статуса заказа

PRODUCT
1. Можно получить информацию о всех продуктах (product)
2. Можно получить информацию и редактировать продукт конкретного пользователя(product/info)

SHOP
1. Можно получить список всех маагазинов (shop)
2. Можно получить список всех категорий (category)

BASET
1. Можно создать заказ со статусом basket и добавлять/удалять различные товары (basket)
2. Затем передать заказ поставщику. При этом выполнится проверка на наличие товара у поставщика в нужном колличестве,
и при положительном результате, статус заказа изменится с basket на new. Пользователю будет отправлен токен подтверждения заказа.
После его активации (user/register/confirm), при получении поставщиком информации о заказе статус заказа изменится на confirmed
